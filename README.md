# Customer Segmentation
**Machine Learning II · Nova IMS · Data Science Degree**

---

## Project Overview

Unsupervised customer segmentation of 33,038 retail customers using K-Means clustering, with basket association rules (Apriori) used to design targeted marketing campaigns for each identified segment.

Five segments were identified:

| Segment | Customers | Avg Lifetime Spend |
|---|---|---|
| High-Value Families | 6,952 | EUR 42,714 |
| Core Everyday Shoppers | 13,458 | EUR 19,235 |
| Pet and Home Essentials | 6,976 | EUR 17,321 |
| Tech-Savvy Singles | 3,324 | EUR 23,739 |
| Budget-Conscious Shoppers | 2,328 | EUR 10,581 |

## Running the Project

### Step 1 — Run the pipeline

Preprocesses the data, fits K-Means (k=5), assigns segment labels, generates all visualisations, mines association rules, and writes `outputs/customer_clusters.csv`.

```bash
python run_pipeline.py
```

Expected runtime: ~5–10 minutes (association rule mining is the slow step).

All outputs are written to `outputs/`. The pipeline logs progress at each step and will complete even if the association rules step encounters an error.

### Step 2 — Build the PDF report

Reads all assets from `outputs/` and writes `customer_segmentation_report.pdf` to the project root.

```bash
python report.py
```

`run_pipeline.py` must be run first so that the figures and rule CSVs exist.

## Methodology

| Step | Choice | Rationale |
|---|---|---|
| Scaling | `StandardScaler` | K-Means is distance-based; features must be on the same scale |
| k selection | Elbow + silhouette (k=2..9) | k=5 maximises silhouette score (0.179) and shows clear elbow inflection |
| Initialisations | `n_init=15` | Consistent across both k-search and final model fit |
| Segment labelling | Feature-signature matching | Greedy sequential assignment by defining characteristic; stable across re-runs |
| Basket rules | Apriori, support≥1%, confidence≥25%, lift≥1.1 | Mined separately per cluster for segment-specific relevance |
