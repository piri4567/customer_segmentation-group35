# Customer Segmentation ‚Äî Interactive Dashboard

Companion React app for the **Customer Segmentation** project (Nova IMS ¬∑ ML II ¬∑ Group 35).
The dashboard presents the segmentation results, cluster profiles, association rules and
campaign suggestions interactively, as an alternative to the static report.

> **Built with:** Vite + React + Tailwind CSS + Recharts + PapaParse.

---

## Quick start (from a clean environment)

```bash
cd app
npm install
npm run dev
```

Then open the URL printed by Vite (typically `http://localhost:5173`).

Production build:

```bash
npm run build
npm run preview
```

That's it ‚Äî no backend, no database. All data (CSVs and figures from
`../outputs/`) is bundled as static assets under `public/`.

---

## Sections

| Tab | What you'll find |
|---|---|
| **Overview** | Executive summary with KPIs, customer split (pie chart), revenue contribution (bar chart), key findings, and strategic priorities. |
| **Segments** | Interactive cards for the 5 segments with full profile, defining characteristics, at-a-glance stats and tailored campaigns. |
| **Methodology** | K-Means decisions table, interactive elbow + silhouette curves loaded from `cluster_metrics.csv`, and PCA scatter. |
| **Visualisations** | All figures generated by `run_pipeline.py`, grouped by theme. |
| **Campaigns** | 4 campaigns per segment with rationale, plus the top association rules for the selected cluster (loaded from `rules_cluster_*.csv`). |

## How to refresh the data

The CSVs and PNGs under `public/data/` and `public/images/` are copies of the
files in `../outputs/`. After re-running the Python pipeline
(`python run_pipeline.py` from the repo root), refresh them:

```bash
cp ../outputs/*.csv public/data/
cp ../outputs/*.png public/images/
```
